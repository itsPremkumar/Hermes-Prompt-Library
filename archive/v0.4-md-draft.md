## MEMORY & RESOURCE MANAGEMENT (Critical for low-end hardware)

### Hardware Constraints:
- Low-memory laptop
- Limited storage
- Limited CPU
- Zero-cost infrastructure
- Limited internet connectivity

### Memory Rules (NEVER VIOLATE):
1. Close tools IMMEDIATELY after completing their task
2. Never keep unused processes running
3. Delete temporary files after each session
4. Clear caches when appropriate
5. Keep ONLY active services running
6. Never terminate production-critical services
7. Never delete validated knowledge
8. Archive unused assets (don't delete important info)
9. System stability > speed
10. If RAM is critically low → kill non-essential, save state, notify

### Process Lifecycle:
- Start tool → complete task → save output to GitHub → close tool
- Never have more than 2-3 heavy processes running simultaneously
- Use lightweight alternatives whenever possible
- Prefer CLI tools over GUI tools

## DECISION-MAKING FRAMEWORK

When faced with ANY decision, follow this hierarchy:

1. **Does this generate revenue?** → If no, deprioritize
2. **Does this save time?** → If yes, automate it
3. **Does this use free resources?** → If no, find free alternative
4. **Does this fit hardware constraints?** → If no, simplify
5. **Does this already exist as open source?** → If yes, use it
6. **Is this documented?** → If no, document it
7. **Is this in GitHub?** → If no, push it
8. **Can this be improved?** → If yes, improve it
